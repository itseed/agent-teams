# Accessibility — contrast / focus / touch / semantics

## Contrast (WCAG)

| ระดับ | อัตราส่วนขั้นต่ำ |
|-------|------------------|
| Text ปกติ (< 18pt / < 14pt bold) | **4.5:1** (AA) |
| Large text (≥ 18pt / ≥ 14pt bold) | **3:1** (AA) |
| UI component / icon ที่สื่อความหมาย, border ของ input | **3:1** |
| AAA (ถ้าต้องการเข้มกว่า) | 7:1 / 4.5:1 |

- **ตรวจทุกคู่ text/background** รวม state (hover, disabled, placeholder) และทุก theme (light + dark)
- placeholder ที่จาง ๆ มักไม่ผ่าน — อย่าใช้ placeholder แทน label
- ตรวจด้วย tool (contrast checker) อย่าเดาด้วยตา; ระบุค่าที่วัดได้ใน spec

## Focus

- ทุก interactive element ต้องมี **focus indicator มองเห็นได้** (อย่า `outline: none` ทิ้ง)
- focus order ตามลำดับการอ่าน (logical), ไม่ข้ามไปมา
- modal/drawer ต้อง trap focus + คืน focus กลับที่เดิมเมื่อปิด

## Touch / target size

- touch target **≥ 44×44px** (iOS) / 48×48dp (Android) — ปุ่มเล็กให้เพิ่ม hit area แม้ icon เล็ก
- ระยะห่างระหว่าง target พอไม่ให้กดพลาด

## Semantics

- ใช้ element ที่ถูกความหมาย (`<button>` ไม่ใช่ `<div onClick>`); heading เป็นลำดับ (h1→h2→h3)
- รูป/icon ที่สื่อความหมายต้องมี `alt` / `aria-label`; icon ตกแต่งล้วน → `aria-hidden`
- form: ทุก input มี `<label>` ผูกจริง; error ผูกด้วย `aria-describedby`
- empty/error state: ข้อความต้องอ่านได้ด้วย screen reader (ไม่ใช่สื่อด้วยสี/icon อย่างเดียว)
- ไม่สื่อความหมายด้วย **สีอย่างเดียว** (เช่น error แดงล้วน) — เพิ่ม icon/ข้อความกำกับ

## Motion
- เคารพ `prefers-reduced-motion` — ลด/ตัด animation ที่ไม่จำเป็นเมื่อผู้ใช้ตั้งค่าไว้

## Checklist
- [ ] ทุกคู่ text/bg ผ่าน contrast AA (วัดจริง, ทุก theme + state)
- [ ] focus indicator มองเห็นทุก interactive element
- [ ] touch target ≥ 44px, ปุ่ม icon มี aria-label
- [ ] ไม่สื่อความหมายด้วยสีอย่างเดียว
- [ ] form input มี label ผูกจริง + error ประกาศได้
